# Contributing to Solum-AI

Solum-AI is currently in foundation stage. Contributions should protect focus and avoid premature complexity.

## Contribution priorities

Good early contributions:

- clearer documentation;
- evaluation tasks;
- scoring rules;
- examples of assistant failures;
- small scripts for repeatable tests;
- safety and quality rules.

Avoid early contributions that add:

- cryptocurrency or financial promises;
- distributed training infrastructure;
- autonomous agents with write/release power;
- broad claims without evaluation;
- large model training before eval exists.

## Pull request rules

A pull request should include:

- what changed;
- why it changed;
- how it can be checked;
- risks or limitations.

## Review principles

Review should focus on:

- clarity;
- correctness;
- scope control;
- safety;
- verifiability.

## Tone

Keep the project technical, calm, and useful. Avoid hype claims and political framing.
