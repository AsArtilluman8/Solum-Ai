# Solum-AI Roadmap

This roadmap intentionally starts small. The goal is to avoid early overreach and build a reliable foundation first.

## Stage 0 — Foundation documents

Status: in progress

Deliverables:

- README;
- vision;
- quality rules;
- workflow;
- evaluation plan;
- contributing rules;
- security policy.

Success criteria:

- the project has a clear public direction;
- the first scope is narrow;
- risky future ideas are documented but not implemented too early.

## Stage 1 — Behavior benchmark

Deliverables:

- discipline eval tasks;
- coding eval tasks;
- verification eval tasks;
- simple scoring format.

Success criteria:

- Solum can compare different prompts/models on reliability and completion behavior.

## Stage 2 — Prompt baseline

Deliverables:

- Solum system prompt draft;
- baseline tests against selected open models and commercial assistants for comparison;
- report with strengths and failures.

Success criteria:

- measurable behavior baseline before any fine-tuning.

## Stage 3 — Coding benchmark

Deliverables:

- small practical coding tasks;
- expected solutions/tests;
- report format.

Success criteria:

- the project can measure code-task completion, not just text quality.

## Stage 4 — Dataset and LoRA experiments

Deliverables:

- small curated behavior dataset;
- training config draft;
- first experimental LoRA;
- before/after eval report.

Success criteria:

- honest result: improvement, no change, or regression.

## Stage 5 — App prototype

Future stage.

Deliverables:

- simple local/web UI;
- project file context;
- command/checklist workflow;
- no unsafe autonomous execution by default.

## Stage 6 — Community and credits

Future stage.

Deliverables:

- contribution points, not cryptocurrency;
- validation tasks;
- reputation roles;
- transparent rules.

## Stage 7 — Model distribution and specialized models

Future stage.

Deliverables:

- local model packages;
- model cards;
- optional P2P distribution;
- specialized models for coding, chat, vision, and generation.
